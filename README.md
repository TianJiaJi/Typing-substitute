# 模拟打字程序 Pro

> 解决某些场景下禁止粘贴的问题，帮你自动"手敲"文本。

## 简介

在一些考试系统、远程桌面、游戏或特定软件中，粘贴功能会被禁用。本程序通过模拟真实的键盘输入，将文本逐字符"打"进目标窗口，绕过粘贴限制。

## 功能特性

- 🚀 **三种输入引擎**
  - **Unicode 注入（推荐）**：通过 Windows `SendInput` API 注入字符，支持中文；失败时自动降级为 `WM_CHAR` 消息，确保中文也能打出
  - **剪贴板粘贴**：逐字符通过剪贴板 + `Ctrl+V` 输入，兼容性较好
  - **物理按键（专治禁止粘贴）**：使用 `keyboard` 库模拟物理键盘事件，适用于最严格的场景
- ⏱️ **速度可调**：支持调节打字速度（0.02s ~ 0.5s/字符）和随机波动，模拟真人打字节奏
- 📂 **文件导入**：支持从 `.txt` 文件加载文本内容
- 📋 **运行日志**：实时显示打字进度和运行状态
- ⏳ **倒计时启动**：3 秒倒计时，方便切换到目标窗口
- 🛑 **随时停止**：打字过程中可随时中断
- 🖥️ **DPI 感知**：支持高 DPI 屏幕，界面不会模糊或缩水

## 环境要求

- **操作系统**：Windows（推荐 Windows 10/11）
- **Python**：3.7+

## 快速开始

1. 克隆或下载项目：
   ```bash
   git clone <repo-url>
   cd Typing-substitute
   ```

2. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```

3. 运行程序（建议使用管理员权限）：
   ```bash
   python main.py
   ```

## 使用方法

1. 在文本框中输入或从文件加载要输入的内容
2. 选择输入引擎（默认推荐 **Unicode 注入**）
3. 调节打字速度和波动幅度
4. 点击 **▶ 开始打字**，在 3 秒倒计时内切换到目标窗口
5. 程序将自动逐字符输入文本
6. 如需中断，点击 **⏹ 停止**

## 常见问题

| 问题 | 解决方案 |
|------|---------|
| 中文打不出来 | 切换引擎为「Unicode 注入」，程序会自动降级到 WM_CHAR |
| 某些窗口完全无反应 | 以管理员权限运行程序 |
| 64 位系统注入失败 | 已内置修复，使用修复后的结构体对齐 |
| 禁止粘贴的场景 | 使用「物理按键」引擎 |
